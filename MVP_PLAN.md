# ArgusV MVP Plan — 2 Days

**MVP Definition:** A deployable surveillance system with live detection, real-time alerts, incident logging, recording playback, zone management, and semantic search — running via Docker Compose.

**Scope:** Detection pipeline, VLM inference, JWT auth, WebSocket alerts, incidents API, zones API, cameras API, Slack notifications, text embeddings (VLM description → pgvector), semantic search.

**Cut from MVP:** CLIP embeddings, PTZ, MQTT actuation, WebPush, birdseye view, Prometheus metrics, RAG chat.

---

## Embedding Approach

Use the VLM's text description (already generated by GPT-4o) → embed via `text-embedding-3-small` (OpenAI) → store in existing `Detection.vlm_embedding` pgvector column → semantic search via cosine similarity.

```
VLM generates description
  → "Person in red jacket loitering near entrance, carrying bag"
    → OpenAI text-embedding-3-small API call
      → 1536-dim vector stored in Detection.vlm_embedding (pgvector)
        → /api/search?q="red jacket" → cosine similarity → matching incidents
```

## Risk Items

1. **HLS recording** — highest risk, FFmpeg subprocess management and segment paths need careful wiring
2. **HLS.js in Next.js** — SSR/client boundary issues, use `dynamic(() => ..., { ssr: false })`
3. **WebSocket auth** — if `DEV_AUTH_BYPASS=false`, WS connection needs token in query param
4. **CORS** — frontend at `localhost:3000` → backend at `localhost:8000` must be in `CORS_ALLOW_ORIGINS`
5. **pgvector** — ensure `CREATE EXTENSION IF NOT EXISTS vector` runs in migration before embedding insert
